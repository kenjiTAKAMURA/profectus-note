## 1. エグゼクティブサマリー

### 1.1 プロジェクト概要

「Profectus Note（プロフェクタスノート）」は、**中小企業の社長・経営層向け**に、IT・セキュリティを中心とした**今すぐアクションに移せる実践的な情報**を発信するナレッジブログです。

社名「Profectus（進歩）」の理念を体現し、**中小企業の日々の進歩を実現するノート**として、企業の成長と発展を支援します。

### 1.2 ブログのミッション

> **中小企業が進歩・発展し、日本のより多くの人の心を豊かにし、みんなが幸せと感じる社会の実現に貢献する**

### 1.3 ブログの位置づけ

```
┌─────────────────────────────────────────────────────────────────┐
│                    情報のインプット（Inbox）                      │
├─────────────────────────────────────────────────────────────────┤
│  書籍 │ SNS │ ネットニュース │ セミナー │ 業界動向 │ 顧客相談     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    自分で理解・解釈・咀嚼                         │
│            専門家の視点で本質を見極め、実践的に再構成              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                Profectus Note（アウトプット）                    │
│                                                                 │
│  ・中小企業の経営層・従業員向けに咀嚼して発信                      │
│  ・次のアクションに活かしてもらえる内容                           │
│  ・日々忙しい中小企業の代わりに情報収集・蓄積                      │
│  ・直接コンサルできない企業への横展開・情報提供                    │
└─────────────────────────────────────────────────────────────────┘
```

**本ブログの役割**
- **攻め（IT・DX）と守り（セキュリティ）のテキスト・参考書**
- 日々忙しい中小企業の経営層・従業員の「代わり」に情報収集・蓄積
- 勘所を伝え、次のアクションに活かしてもらう
- 自分のナレッジ蓄積であり、かつコンテンツとして横展開可能な資産

### 1.4 差別化要素

| 競合との差別化ポイント | 詳細 |
|---|---|
| **中小企業社長目線** | 「自分が社長なら欲しい情報」を起点に記事を作成 |
| **即アクション可能** | 読んですぐ実行に移せる具体的なステップを提示 |
| **攻めと守りの両面** | IT・DX（攻め）とセキュリティ（守り）を統合的にカバー |
| **統合的視点** | 経営戦略・IT・セキュリティ・財務を横断的に捉える |
| **実務経験** | 大手SIer・電力会社での実務経験に基づく現場感覚 |
| **資格裏付け** | 中小企業診断士・情報処理安全確保支援士・日商簿記1級・FP1級 |
| **地域特化** | 富山県という地方の文脈に根ざしたアドバイス |

### 1.5 注意事項：個人雑記ブログにしない

本ブログは以下の点を厳守し、**ビジネス特化のプロフェッショナルメディア**として運営します。

**掲載しない内容**
- 個人の日常や私生活に関する投稿
- 趣味や嗜好に関する雑記
- 感想だけで終わる記事
- 中小企業経営に関係のない時事ネタ

**常に問う基準**
- 「自分が中小企業の社長なら、この情報が欲しいか？」
- 「読者がこの記事を読んで、明日から何かアクションに移せるか？」
- 「中小企業の経営・業務に直接役立つか？」

### 1.6 期待効果

- **認知拡大**: 富山県内のDX支援ニーズを持つ企業・自治体へのリーチ
- **信頼構築**: 専門性の発信による戦略顧問としてのブランディング
- **見込客獲得**: 記事を通じた問い合わせ・相談の獲得
- **ナレッジ蓄積**: 自身の知見の体系化と資産化
- **横展開**: 直接コンサルできない企業への情報提供

## 2. 市場環境分析

### 2.1 2026年 DX市場の動向

2026年現在、DX市場は以下の特徴を示しています：

**マクロトレンド**
- DXコンサルティング市場規模：年率10%成長で2029年には1兆2,832億円に到達予測（IDC Japan）
- 高市政権による「DX 3分野」（建築・交通・製造）への国策投資
- AIエージェント元年（2025年）を経て、「導入」から「仕組み化」フェーズへ移行

**中小企業のDX現状**
- DXに成功している中小企業：全体の**わずか4.6%**（経済産業省 DX支援ガイダンス 2024）
- 約3分の2の中小企業が業務プロセスのデジタル化に未着手
- 主な課題：人材不足、IT知識の欠如、投資余力の不足

### 2.2 富山県のDX環境

**支援制度**
- 富山県中小企業トランスフォーメーション補助金（最大500万円、補助率3/4）
- IT導入補助金2025
- 富山県成長戦略ビジョン「幸せ人口1000万～ウェルビーイング先進地域、富山～」

**地域課題**
- 製造業中心の産業構造（DX投資の潜在需要大）
- 中小企業が主体で、大企業のようなIT部門を持たない
- IT人材の都市部流出

### 2.3 競合分析

| カテゴリ | 主な競合 | 強み | 弱み |
|---|---|---|---|
| **大手コンサルブログ** | アクセンチュア、ベイカレント等 | ブランド力、リソース | 地方視点の欠如、敷居の高さ |
| **診断士個人ブログ** | 各地の中小企業診断士 | 親しみやすさ | IT専門性の不足 |
| **IT系メディア** | ITmedia、@IT等 | 情報量、速報性 | 地方中小企業目線の欠如 |
| **SaaS提供企業ブログ** | Salesforce、freee等 | 製品連動 | 中立性への疑念 |

**Profectus Noteのポジショニング**
→ 「中小企業の社長が、IT・セキュリティの勘所を学び、明日からアクションに移せる実践的参考書」

### 3.1 ブログタイトル・タグライン

### 3.2 ブログの位置づけ

```

    │

          ↑
          │ リンク・誘導
          │

    │
    ├── 【攻め】IT・DX戦略記事
    ├── 【守り】セキュリティ解説
    ├── 社長インタビュー・事例紹介（将来展開）
    ├── 最新ニュース解説
    └── 補助金・支援制度情報
```

### 3.3 コンテンツの基本方針

**1. 中小企業社長の視点で考える**
- 「自分が中小企業の社長なら、どういう情報が欲しいか」を常に問う
- 経営者が抱える具体的な悩みからコンテンツを設計

**2. 今からアクションに移せる内容**
- 読んで「なるほど」で終わらせない
- 必ず「次に何をすればいいか」を明示
- 具体的なステップ、チェックリスト、テンプレートを提供

**3. 攻め（IT・DX）と守り（セキュリティ）の両輪**
- DX・IT活用による成長戦略（攻め）
- セキュリティ対策によるリスク管理（守り）
- 両方のバランスを取った経営の勘所を伝える

**4. テキスト・参考書的な扱い**
- 体系的に整理された知識体系
- 必要な時に必要な情報を引ける構成
- 継続的に参照される永続的なコンテンツ

### 3.4 デザインコンセプト

**キーワード**: 信頼感、専門性、親しみやすさ、実践的

**カラースキーム**:
- プライマリ: ディープブルー (#1a365d) - 信頼・専門性
- セカンダリ: アクセントグリーン (#38a169) - 成長・進歩
- ベース: クリーンホワイト (#f7fafc)

**フォント**:

### 6.2 CMS選定

**推奨: Hugo または Astro + ヘッドレスCMS（microCMS）**

| 評価項目           | Hugo/Astro           | WordPress      |
| -------------- | -------------------- | -------------- |
| **Obsidian連携** | ◎ Markdownネイティブ      | △ プラグイン必要      |
| **表示速度**       | ◎ 静的サイト、高速           | ○ キャッシュで対応可    |
| **セキュリティ**     | ◎ 攻撃対象が少ない           | △ 継続的なアップデート必要 |
| **運用コスト**      | ◎ Cloudflare Pages無料 | ○ サーバー費用必要     |
| **カスタマイズ性**    | ○ テーマ・コード知識必要        | ◎ プラグインで拡張容易   |
| **情報量**        | ○ 十分だが英語が多い          | ◎ 日本語情報豊富      |
| **将来のHP連携**    | ○ 別途構築               | ◎ マルチサイト対応     |

**最終推奨**: 
- 技術志向を強調するなら **Astro + microCMS**
- 運用の容易さを優先するなら **WordPress + Xserver**

### 6.3 ホスティング

**静的サイト（Hugo/Astro）の場合**

## 8. n8n自動化ワークフロー

### 8.1 n8n概要と導入方法

**n8nとは**
オープンソースのワークフロー自動化ツール。ローカル実行なら無料で利用可能。700以上のサービスと連携でき、AIとの統合も容易です。

**導入方法（Docker推奨）**

```yaml
# docker-compose.yml
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - GENERIC_TIMEZONE=Asia/Tokyo
      - TZ=Asia/Tokyo
    volumes:
      - n8n_data:/home/node/.n8n
      - /path/to/obsidian/vault:/obsidian
    networks:
      - n8n-network

volumes:
  n8n_data:

networks:
  n8n-network:
    driver: bridge
```

### 8.2 ワークフロー1: RSS→Obsidian自動取り込み

**目的**: 最新ニュースサイトからRSSを読み込み、Obsidianにノートとして保存

```
[RSS Feed Trigger]
    │ (毎日 or 6時間ごと)
    ▼
[HTTP Request]
    │ (記事本文を取得)
    ▼
[HTML Extract]
    │ (タイトル、本文、日付を抽出)
    ▼
[AI Agent (Claude API)]
    │ (要約、タグ抽出、カテゴリ分類)
    │ (中小企業社長にとっての重要度を判定)
    ▼
[Code Node]
    │ (Markdown形式に整形)
    ▼
[Convert to File]
    │ (バイナリ化)
    ▼
[Read/Write Files from Disk]
    │ (Obsidian/01_Inbox/RSS/[category]/に保存)
    ▼
[完了]
```

**取り込み対象RSS例**
- ITmedia NEWS (https://rss.itmedia.co.jp/rss/2.0/news_bursts.xml)
- Security NEXT (https://www.security-next.com/feed)
- 経済産業省 新着情報 (https://www.meti.go.jp/information_2/news_rss_new.xml)

    │ (公開完了通知)
    ▼
[完了]
```

## 9. Cursor・MDCファイル活用

### 9.1 Cursorとは

Cursorは、AIを搭載したコードエディタで、VS Codeをベースにしています。`.mdc`ファイル（Markdown with Context）を使用することで、AIの動作をプロジェクトごとにカスタマイズできます。

### 9.2 ブログ執筆におけるCursor活用

**メリット**
- AIがプロジェクトのルール・文体を学習
- 一貫した品質の記事作成をサポート
- Obsidianとの相性（Markdown対応）

**運用フロー**
```
1. Obsidianでアイデア・素材を蓄積
2. CursorでObsidian Vaultを開く
3. .mdcルールに従ってAIが記事作成支援
4. 完成した記事を05_Output/ProfectusNote/に保存
5. n8nでブログへ自動公開
```

### 9.3 MDCファイル設計

**ディレクトリ構造**
```
Obsidian Vault/
└── .cursor/
    └── rules/
        ├── me.mdc              # 著者の人格・特徴
        ├── writing-rule.mdc    # ライティングルール
        ├── blog-post.mdc       # ブログ記事作成ルール
        ├── content-filter.mdc  # コンテンツフィルター（雑記防止）

```

### 9.4 MDCファイル内容（詳細は付録参照）

- 記事の構成パターン
- 使用する/しない表現

- SEOキーワードの配置ルール

**content-filter.mdc の役割（新規追加）**
- 個人雑記コンテンツのフィルタリング
- 中小企業経営に関係ない内容の排除
- 「自分が社長なら欲しいか」のチェック

以下のコンテンツは本ブログに掲載しない：

### 絶対NG
- 個人の日常や私生活に関する投稿
- 趣味や嗜好に関する雑記（旅行、グルメ、趣味など）
- 感想だけで終わる記事（「面白かった」「勉強になった」のみ）
- 中小企業経営に直接関係のない時事ネタ
- 自分語り、自慢話

### 要注意（掲載する場合は必ず経営との関連を明示）
- 読んだ本の紹介（→ 中小企業経営にどう活かせるかを必ず書く）
- 参加したセミナーの報告（→ 社長が知るべきポイントを抽出）
- 個人的な気づき（→ 中小企業での実践例を必ず添える）

## 公開前の3つのチェック

記事を公開する前に、以下の3つの質問にすべて「Yes」と答えられるか確認する：

1. **「自分が中小企業の社長なら、この情報が欲しいか？」**
   - 忙しい社長が時間を割いて読む価値があるか

2. **「読者がこの記事を読んで、明日から何かアクションに移せるか？」**
   - 具体的な次のステップが明示されているか

3. **「中小企業の経営・業務に直接役立つか？」**
   - 経営課題の解決に貢献するか

## 記事作成時の心構え

- 「自分が書きたいこと」ではなく「読者（社長）が知りたいこと」を書く
- 「面白い話」ではなく「役に立つ話」を優先する
- 「知識のひけらかし」ではなく「実践への橋渡し」を目指す

# Profectus Note ライティングルール

## 1. 記事構成

### 基本構成
1. **リード文**（100-150字）: 読者の課題を明示し、記事を読むメリットを提示
2. **目次**（自動生成）
3. **本文**（見出しごとに300-500字）
4. **★今日からできるアクション**（必須）: 具体的なアクション3つ以上
5. **まとめ**（100-200字）: キーポイントの再確認
6. **CTA**（任意）: 関連記事、問い合わせ誘導

### 見出しルール
- H2: 大きなセクション（3-5個）
- H3: H2の中の小項目（各H2に2-4個）
- H4以降: 極力使わない（使う場合はH3の補足）

### ★「今日からできるアクション」の書き方
- 必ず3つ以上のアクションを箇条書きで提示
- 「〜を確認する」「〜に相談する」「〜を導入する」など動詞で始める
- 所要時間の目安があると親切（例：「所要時間：10分」）
- 難易度別に分けると読者が選びやすい（例：「すぐできる」「今週中に」「今月中に」）

## 2. 文体・トーン

### 基本トーン
- 「です・ます」調
- 専門家として信頼感を持ちつつ、親しみやすく
- 上から目線にならない
- 社長（読者）を尊重する姿勢

### 使用する表現
- 「〜が重要です」「〜をおすすめします」
- 「〜ではないでしょうか」（問いかけ）
- 「〜という声をよく聞きます」（共感）
- 「社長の皆さまにとって」「経営者として」（読者への語りかけ）

### 避ける表現
- ❌ 「〜すべきです」（強制的）
- ❌ 「〜しなければなりません」（義務的）
- ❌ 「当然ですが」「ご存知の通り」（読者を選ぶ）
- ❌ 過度なカタカナ・英語
- ❌ 「私は〜」「私の経験では〜」の多用（自分語り）

## 3. 専門用語の扱い

- 初出時に必ず説明を添える
  - 良い例: 「DX（デジタルトランスフォーメーション：デジタル技術を活用した事業変革）」
- 略語は正式名称を併記
- 技術用語はできるだけ平易な言い換えも提示
- 「社長が部下に説明できるレベル」を意識

## 4. SEO配慮

### タイトル
- 文字数: 28-35文字
- キーワードを前方に配置
- 数字、疑問形を効果的に使用
  - 例: 「【2026年版】中小企業がDXを成功させる5つのステップ」

### メタディスクリプション
- 文字数: 80-120文字
- キーワードを自然に含める
- 記事の価値を端的に伝える

### 本文
- H2にキーワードを含める
- 共起語を意識的に配置
- 適切な内部リンク、外部リンク

## 5. 引用・出典

- 公的機関のデータ優先（経済産業省、IPA、総務省等）
- 出典元を明記
- 引用は必要最小限、オリジナルの解釈・解説を加える

## 6. 文字数目安

| 記事タイプ | 文字数 |
|---|---|
| ニュース解説 | 1,500-2,000字 |
| 専門記事 | 2,500-4,000字 |
| 柱記事（ガイド） | 5,000-8,000字 |
| インタビュー記事 | 3,000-5,000字 |
| 事例紹介 | 2,000-3,000字 |

## 7. 禁止事項

- 他サイトのコピー・無断引用
- 誇大表現、根拠のない断言
- 特定企業・個人の誹謗中傷
- ポジショントークのみの記事
- 個人的な雑感だけの記事
- アクションアイテムのない記事
```
