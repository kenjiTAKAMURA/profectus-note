## ブログのコンセプト

### ミッション

> **中小企業が進歩・発展し、日本のより多くの人の心を豊かにし、みんなが幸せと感じる社会の実現に貢献する**

### ブログの位置づけ

**Profectus Note**は、中小企業の社長・経営層向けに、IT・DX（攻め）とセキュリティ（守り）を中心とした**今すぐアクションに移せる実践的な情報**を発信するナレッジブログです。

### 差別化要素

| ポイント | 詳細 |
|---------|------|
| **中小企業社長目線** | 「自分が社長なら欲しい情報」を起点に記事を作成 |
| **即アクション可能** | 読んですぐ実行に移せる具体的なステップを提示 |
| **攻めと守りの両面** | IT・DX（攻め）とセキュリティ（守り）を統合的にカバー |
| **統合的視点** | 経営戦略・IT・セキュリティ・財務を横断的に捉える |
| **実務経験** | 大手SIer・電力会社での実務経験に基づく現場感覚 |
| **資格裏付け** | 中小企業診断士・情報処理安全確保支援士・日商簿記1級・FP1級 |
| **地域特化** | 富山県という地方の文脈に根ざしたアドバイス |

## ページ構成

### トップページ（`/`）

**構成:**
1. ヒーローセクション
   - サイトタイトル「Profectus Note」
   - タグライン「中小企業の攻めと守りを支える、実践ナレッジ」
   - サブタイトル（ブログの説明）
2. コンセプト説明
   - 攻め（IT・DX）カード
   - 守り（セキュリティ）カード
3. このブログの特徴
   - 差別化要素を箇条書きで表示
4. 最新記事
   - 最新3件の記事を表示
   - 「すべての記事を見る」リンク

### ブログ一覧ページ（`/blog`）

**構成:**
- 記事一覧（グリッドレイアウト）
- 各記事カードに表示される情報：
  - ヒーロー画像
  - 記事タイトル
  - カテゴリバッジ
  - 攻め/守りバッジ
  - 公開日

### ブログ記事ページ（`/blog/[slug]/`）

**構成:**
1. ヒーロー画像（記事のトップ）
2. メタ情報セクション
   - カテゴリバッジ
   - 攻め/守りバッジ
   - 公開日・更新日
3. タイトルと説明文
4. 記事本文
5. 今日からできるアクションセクション（actionItemsがある場合）
   - グラデーション背景
   - チェックマーク付きリスト
   - 強調表示

## 記事構成テンプレート

### 基本構成

```
1. リード文（100-150字）
   - 読者の課題を明示
   - この記事を読むメリットを端的に伝える

2. 本文
   - 背景・なぜ重要か
   - 具体的な方法・ステップ
   - 注意点・よくある失敗

3. ★今日からできるアクション（必須）
   - 読んですぐ実行できる具体的なアクション
   - チェックリスト形式が望ましい
   - 3つ以上提示

4. まとめ
   - キーポイントの再確認
   - 関連記事への誘導
```

### Frontmatterテンプレート

```yaml
---
title: "記事タイトル"
description: "記事の説明文（100-150字程度）"
pubDate: 2026-02-06
updatedDate: 2026-02-06  # オプション
category: "dx"  # 必須
tags:
  - "DX"
  - "中小企業"
attackOrDefense: "attack"  # "attack", "defense", "both"
heroImage: "/images/blog/article-name/hero.jpg"
draft: false
actionItems:  # オプション（Frontmatterで指定すると自動表示）
  - "アクション1"
  - "アクション2"
  - "アクション3"
---
```

## コンテンツ方針

### 掲載する内容

- 中小企業の経営・業務に直接役立つ情報
- 読者が明日からアクションに移せる具体的なステップ
- IT・DX（攻め）とセキュリティ（守り）に関する実践的なノウハウ
- 補助金・支援制度の解説
- 最新ニュースの解説（中小企業への影響を明示）

### 掲載しない内容

- 個人の日常や私生活に関する投稿
- 趣味や嗜好に関する雑記
- 感想だけで終わる記事
- 中小企業経営に関係のない時事ネタ
- 自分語り、自慢話

### 記事作成時のチェックポイント

1. 「自分が中小企業の社長なら、この情報が欲しいか？」
2. 「読者がこの記事を読んで、明日から何かアクションに移せるか？」
3. 「中小企業の経営・業務に直接役立つか？」

## 運用方針

### 投稿頻度

- **目標**: 週3-4記事
- **内訳**:
  - 自動生成（ニュース解説）: 週2本
  - オリジナル記事: 週1-2本

### 記事の種類

1. **柱記事（Pillar Content）**: 参考書の「章」に相当する体系的コンテンツ
2. **クラスター記事**: 柱記事から派生した実践的な記事
3. **ニュース解説**: 最新動向を中小企業目線で解説
4. **事例紹介**: 成功事例・失敗事例の共有（将来展開）

### 品質管理

**公開前チェックリスト:**
- [ ] 3つのチェックポイントをクリアしているか
- [ ] 「★今日からできるアクション」が3つ以上あるか
- [ ] 専門用語に説明が添えられているか
- [ ] 出典・参照元が明記されているか
- [ ] 誤字脱字がないか
- [ ] Frontmatterが正しく設定されているか

## 技術仕様

### 使用技術

- **フレームワーク**: Astro 5.17.1
- **言語**: TypeScript
- **スタイリング**: CSS（カスタムプロパティ）
- **コンテンツ管理**: Astro Content Collections（Markdown/MDX）

### パフォーマンス最適化

**実装済みの最適化:**
- CSSの完全インライン化（レンダリングブロック回避）
- Google Fontsの非同期読み込み
- フォント読み込みの最適化（Font Loading API）
- リフローの最小化（requestAnimationFrame）
- 共通スタイルの統合（global.cssに集約）
- レスポンシブ対応（モバイル・タブレット）

### ダークモード/ライトモード

- ヘッダーにテーマ切り替えボタンを配置
- localStorageに保存（次回訪問時に適用）
- システム設定（prefers-color-scheme）に対応
- リフローを最小化した実装

### デプロイ

- **ホスティング**: Cloudflare Pages
- **ドメイン**: profectus-note.com
- **SSL**: 自動設定
- **CDN**: Cloudflare CDN

### 自動化

- **Git連携**: GitHubにpushすると自動デプロイ

- **サイトマップ**: 自動生成（`/sitemap-index.xml`）
